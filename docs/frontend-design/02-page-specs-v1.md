# 第一版页面说明

本文档定义第一版优先设计和开发的前端页面。

## 路由总览

登录前：

- `/login`
- `/register`，可选，第一版可作为申请账号或占位页

登录后：

- `/app/home`
- `/app/groups`
- `/app/groups/:groupId`
- `/app/groups/:groupId/admin`
- `/app/group-buys/new`
- `/app/group-buys/:groupBuyId/edit`
- `/app/group-buys/:groupBuyId`
- `/app/goods`
- `/app/goods/:goodsId`
- `/app/me`
- `/app/me/records`
- `/app/me/charges`
- `/app/me/dispatchable-items`
- `/app/me/payment-channels`
- `/app/me/addresses`
- `/app/dispatch-requests/new`

后续流程页：

- `/app/warehouse`
- `/app/warehouse/dispatch-requests`
- `/app/warehouse/dispatch-requests/:dispatchRequestId`
- `/app/international-batches/new`
- `/app/international-batches/:batchId`
- `/app/admin`
- `/app/admin/transfers`
- `/app/admin/exceptions`
- `/app/admin/users`
- `/app/admin/audit-logs`

说明：

- 详细业务页面入口和弹窗承载规则见 [07-business-page-map.md](07-business-page-map.md)。

## 登录页

路径：

- `/login`

目标：

- 让用户进入系统。

主要字段：

- 账号或 QQ 号
- 密码

主要操作：

- 登录
- 跳转注册或申请账号

接口需求：

- `POST /api/auth/login`
- `GET /api/app/bootstrap`

## 注册页

路径：

- `/register`

目标：

- 作为登录页的次级入口，用于创建普通成员账号、提交加入申请，或在第一版暂时显示申请说明。

主要字段：

- 展示名
- QQ 号
- 群昵称
- 密码
- 确认密码

主要操作：

- 提交注册
- 返回登录

接口需求：

- `POST /api/auth/register`

说明：

- 第一版不把注册作为核心流程；如果后端暂不开放自由注册，该页只需要能承接登录页入口并给出申请账号说明。

## 首页

路径：

- `/app/home`

第一版目标：

- 作为占位和后续教程入口。

内容：

- 使用教程占位
- 常见问题占位
- 当前用户欢迎信息

说明：

- 第一版不作为核心流程页。

## 谷团列表页

路径：

- `/app/groups`

目标：

- 展示用户可见的谷团。

主要内容：

- 谷团列表
- 谷团名称
- 谷团头像或封面
- 当前进行中的拼团数量
- 当前成员数量
- 用户在该团中的角色

主要操作：

- 点击谷团进入详情

接口需求：

- `GET /api/app/groups`

## 谷团详情页

路径：

- `/app/groups/:groupId`

目标：

- 展示团信息和当前正在进行中的拼谷。

页面区域：

- 顶部标题栏
- 团信息卡片
- 管理入口条
- 拼谷列表区
- 搜索和状态筛选

团信息卡片字段：

- 团名称
- 团头像或默认图
- 创建时间或说明
- 团长或维护人
- 成员数量
- 当前用户角色

管理入口：

- 管理台入口
- 管理团信息按钮
- 管理入口根据角色决定是否显示

拼谷列表字段：

- 拼团主图
- 拼团标题
- 拼团类型
- 拼团状态
- 发起人
- 拼团商品数量
- 当前可拼数量
- 截团时间
- 当前用户是否已参与

拼团卡片版式：

- 上半部分展示拼团主图，缺图时显示默认占位图。
- 下半部分展示信息，优先展示标题、类型、状态、发起人。
- 截团时间、商品数量、可拼数量和我的参与状态作为次级信息。
- 手机端卡片保持上图下信息；PC 端可以在同一信息结构下扩展为更宽的卡片或表格行。

筛选：

- 全部
- 进行中
- 未开始
- 已截团
- 已完成
- 已取消

主要操作：

- 点击拼团进入拼团详情
- 管理员或维护人点击新建拼团
- 管理员点击编辑团信息
- 有权限用户进入管理台

接口需求：

- `GET /api/app/groups/:groupId/home`
- `GET /api/app/groups/:groupId/group-buys`

## 管理台入口页

路径：

- `/app/groups/:groupId/admin`

目标：

- 根据角色展示可用管理能力。

管理员可见模块：

- 团信息管理
- 用户和角色
- 拼团管理
- 拼单记录
- 费用付款
- 转运入库
- 囤货排发
- 转单异常
- 审计日志

拼单维护人可见模块：

- 拼团管理
- 拼单记录
- 费用付款
- 下单截图
- 国际转运
- 转单异常

囤货人可见模块：

- 入库
- 可排发商品
- 排发申请
- 国内运费
- 国内快递

接口需求：

- `GET /api/app/groups/:groupId/admin-capabilities`

## 新建和编辑拼团页

路径：

- `/app/group-buys/new`
- `/app/group-buys/:groupBuyId/edit`

目标：

- 创建或编辑一个拼团。

主要字段：

- 所属谷团
- 拼团类型
- 标题
- 说明
- 截团时间
- 默认收款渠道
- 默认囤货人
- 拼团商品
- 商品价格
- 商品预估重量
- 数量限制
- 内定库存

主要操作：

- 保存草稿
- 发布为拼拼拼
- 添加拼单商品
- 编辑拼团状态
- 选择已有收款方式
- 快速新建收款方式
- 从商品图鉴选择商品
- 手动录入商品
- 发起调价申请

接口需求：

- `POST /api/group-buys`
- `PATCH /api/group-buys/:groupBuyId`
- `POST /api/group-buys/:groupBuyId/items`

说明：

- 商品选择可以引用商品图鉴，也可以手动录入。
- 创建拼单商品时进入库存和快照字段填写。
- 新建和编辑共用同一套表单组件。
- 商品、库存、内定库存和收款方式都在该页面内完成。
- 已生成应收或付款后的价格调整需要进入调价审核流程。

## 拼团详情页

路径：

- `/app/group-buys/:groupBuyId`

目标：

- 展示一个拼团的详情、拼单商品和用户认领入口。

页面区域：

- 拼团基础信息
- 状态和类型
- 拼单商品列表
- 我的记录摘要
- 管理操作区

成员操作：

- 查看商品
- 选择数量
- 认领
- 查看我的状态

维护人操作：

- 编辑拼团信息
- 添加拼单商品
- 修改库存
- 上传下单截图
- 查看拼单记录
- 查看付款记录
- 确认或驳回付款
- 创建国际转运批次入口

弹窗和抽屉：

- 上传付款凭证弹窗，成员从费用卡片打开。
- 付款确认或驳回弹窗，维护人从付款记录列表行打开。
- 上传下单截图弹窗，维护人从管理操作区打开。
- 转单申请弹窗可从我的拼单页面打开，不作为拼团详情主入口。

接口需求：

- `GET /api/app/group-buys/:groupBuyId/detail`
- `POST /api/group-buy-records`
- `PATCH /api/group-buys/:groupBuyId`

## 商品图鉴页

路径：

- `/app/goods`
- `/app/goods/:goodsId`

目标：

- 作为日常商品资料查询和新建拼团选品来源。

列表重点：

- 关键词搜索
- 角色筛选
- 支持多选角色
- 支持包含所选角色
- 支持仅包含所选角色
- 筛选详情小窗可展开和收起

主要操作：

- 查看商品详情
- 登录后有权限用户编辑商品详情
- 在新建和编辑拼团页中作为商品选择小浮窗或抽屉使用

接口需求：

- `GET /api/goods`
- `GET /api/goods/:goodsId`
- `POST /api/goods`
- `PATCH /api/goods/:goodsId`

## 我的拼单和我的记录页

路径：

- `/app/me/records`

入口：

- 我的页面

目标：

- 展示当前用户参与过的拼团、商品份额和履约状态。

主要内容：

- 拼团信息
- 商品份额
- 展示状态
- 首款、国际费和国内运费状态
- 排发状态
- 转单状态

主要操作：

- 查看拼团详情
- 查看关联费用
- 勾选记录
- 打开转单申请弹窗

## 我的费用页

路径：

- `/app/me/charges`

入口：

- 我的页面
- 拼团详情页费用卡片

目标：

- 展示当前用户所有费用。

筛选：

- 全部
- 待付款
- 已提交
- 已确认
- 已驳回
- 退款
- 已取消

主要操作：

- 查看费用详情
- 打开上传付款凭证弹窗
- 跳转关联拼团详情

说明：

- 上传付款凭证优先做弹窗，不单独做页面。

## 收款方式页

路径：

- `/app/me/payment-channels`

入口：

- 我的页面
- 新建和编辑拼团页

目标：

- 维护当前用户的收款二维码、账号、展示名和说明。
- 新建拼团时选择已有收款方式，或快速新建收款方式。

主要操作：

- 新建收款方式
- 编辑收款方式
- 停用收款方式

## 可排发商品页

- `/app/me/dispatchable-items`

- `/app/dispatch-requests/new`

目标：

主要字段：

- 收货人
- 手机号
- 国内地址
- 备注

主要操作：

- 选择地址簿地址

路径：

- `/app/me/addresses`

入口：

- 我的页面
- 创建排发申请页

目标：

- 维护常用收货信息，辅助排发申请填写。

说明：

- 第一版排发申请仍保存地址快照。

## 国际转运添加页

路径：

- `/app/international-batches/new`

入口：

- 拼团详情页管理操作区
- 拼单维护人工作台

目标：

- 创建国际转运批次。

第一版必填：

- 金额
- 关联哪些拼单

后续字段：

- 转运平台
- 批次编号
- 国际物流单号
- 重量
- 税费
- 其他费用
- 国内收货囤货人

## 囤货工作台页

路径：

- `/app/warehouse`

入口：

- 谷团详情页管理台入口
- 管理台入口页

目标：

- 为囤货人集中展示入库、排发申请、国内运费和国内快递处理任务。

主要操作：

- 查看分配给自己的国际转运批次
- 确认国内入库
- 进入排发申请处理
- 导出当前工作台数据

## 排发申请处理页

路径：

- `/app/warehouse/dispatch-requests`
- `/app/warehouse/dispatch-requests/:dispatchRequestId`

入口：

- 囤货工作台页

目标：

- 处理成员提交的排发申请。

主要操作：

- 查看申请明细
- 核对和打包
- 录入国内运费
- 确认国内运费付款
- 录入快递公司和单号
- 标记完成
