# 响应式布局规范

## 目标

前端需要同时适配手机浏览器和 PC 浏览器。

优先级：

1. 手机端好用
2. PC 端能高效测试和管理
3. 同一套路由和业务组件复用

## 断点建议

建议断点：

- `mobile`：小于 768px
- `tablet`：768px 到 1023px
- `desktop`：1024px 及以上

手机端：

- 单列布局
- 底部 Tab
- 列表以卡片为主
- 操作按钮放在卡片内、底部浮动栏或页面底部

PC 端：

- 主内容居中或左侧导航布局
- 列表可以切换为表格
- 筛选器横向展开
- 详情可使用右侧抽屉或独立页面

## App Shell 布局

### 手机端

结构：

- 顶部标题栏
- 内容滚动区
- 底部 Tab

尺寸建议：

- 底部 Tab 高度 64px 到 76px
- 内容区底部预留 Tab 安全距离
- 主要按钮高度 44px 到 48px
- 卡片圆角 8px 到 12px

### PC 端

结构建议：

- 左侧导航宽度 220px 到 260px
- 顶部用户栏高度 56px
- 主内容最大宽度按页面决定

说明：

- PC 端不强制底部 Tab。
- 可以把底部 Tab 转成左侧导航。
- 谷团详情和拼团详情在 PC 端可以展示更多字段和操作按钮。

## 页面容器

手机端：

- 页面左右留白 16px
- 列表卡片间距 12px
- 页面背景使用浅灰或柔和浅色
- 重要卡片使用白底

PC 端：

- 工作台页面左右留白 24px
- 表格页使用更宽容器
- 管理台可使用左右分栏

## 状态与操作呈现

状态标签：

- 状态必须比普通文本更醒目
- 不同状态使用不同颜色，但避免过度鲜艳
- `可排发` 应最明显
- `未肾`、`未补国际`、`未到货` 应能快速扫到

按钮：

- 手机端主要操作按钮不宜过多
- 一张卡片最多突出 1 个主按钮
- 次要操作放入更多菜单或详情页

## 参考图转译

参考图中的结构可转译为：

- 顶部蓝色或浅色头图区
- 团信息卡片包含头像、标题、日期、团长、人数
- 管理入口单独成条，文案类似“工作台 管理团内排单、排发中心等”
- 拼谷列表区域包含搜索框、状态 Tab、列表内容
- 手机底部保留主要导航或快捷操作

注意：

- RibSys2 是业务记录系统，不是活动营销页。
- 界面应更注重信息可扫、状态清晰、操作路径短。
