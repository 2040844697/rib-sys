# 产品结构与 App Shell

## 第一版产品结构

第一版采用移动端轻应用结构。

登录前：

- 登录页
- 注册或申请账号入口

登录后：

- 首页
- 谷团
- 我的

底部常驻 Tab：

- `首页`
- `谷团`
- `我的`

说明：

- 手机端底部 Tab 常驻。
- PC 端可转换成左侧导航或顶部导航，但保留同样的信息架构。
- 首页第一版先放占位内容，后续放使用教程、公告或常见问题。
- 我的第一版主要是个人信息编辑和账号状态展示。
- 谷团是第一版重点。

## 登录与注册

第一版以常规登录界面为主。注册或申请账号不作为首屏核心流程，只在登录页保留一个入口；如果暂时不开放注册，可以先做成占位页或申请账号说明页。

### 登录页

路径建议：

- `/login`

主要元素：

- 系统名称
- QQ 号或账号输入框
- 密码输入框
- 登录按钮
- 注册或申请账号入口，弱化展示

第一版规则：

- 如果后端先做简单账号体系，可使用账号密码登录。
- 后续可以扩展第三方登录或管理员创建账号。
- 登录页不要长期展示固定演示账号或固定密码提示；开发态快捷登录应只在开发模式显示。

### 注册页或申请账号页

路径建议：

- `/register`

主要元素：

- 展示名
- QQ 号
- 群昵称
- 密码
- 确认密码
- 提交按钮

说明：

- 如果第一版不开放自由注册，该页可以改成“申请加入”或简单占位说明。
- 提交后可进入待审核状态，或者直接创建普通成员账号。

## App Shell

App Shell 是登录后的外层布局。

手机端结构：

- 顶部安全区
- 当前页面标题
- 主内容区
- 底部 Tab

PC 端结构：

- 顶部栏或左侧栏
- 当前用户区域
- 主内容区最大宽度适配
- 管理类页面可以使用更宽的表格布局

App Shell 需要持有：

- 当前用户信息
- 当前用户角色
- 当前用户能力列表
- 默认群组
- 全局 loading 和错误提示

## 底部 Tab

### 首页

路径：

- `/app/home`

第一版内容：

- 暂时放占位文案
- 后续放教程、公告、使用说明

### 谷团

路径：

- `/app/groups`

第一版内容：

- 展示可见谷团列表
- 当前只有一个谷团也按列表设计
- 点击谷团进入谷团详情

### 我的

路径：

- `/app/me`

第一版内容：

- 展示个人信息
- 编辑展示名、QQ 号、群昵称
- 查看角色
- 进入我的拼单和我的记录
- 进入我的费用
- 进入可排发商品
- 进入收款方式
- 进入地址簿
- 退出登录

## 谷团详情结构

路径：

- `/app/groups/:groupId`

进入后默认展示当前正在进行中的拼谷。

页面区域：

- 顶部团信息区域
- 管理入口区域
- 当前拼谷列表
- 状态筛选和搜索
- 底部或悬浮操作区

参考图中的可借鉴结构：

- 顶部有明显标题和返回按钮
- 团信息使用圆角卡片承载
- 管理入口独立成一条醒目操作区
- 拼谷列表下方用状态 Tab 切换
- 手机端主要操作放在底部或卡片按钮中

## 角色差异

普通成员：

- 查看谷团
- 查看拼团列表
- 进入拼团详情
- 认领商品
- 查看我的记录
- 上传付款凭证
- 申请排发
- 申请转单
- 维护地址簿
- 维护自己的收款方式

拼单维护人：

- 普通成员能力
- 新建拼团
- 编辑拼团
- 管理拼单商品库存
- 查看拼单记录
- 确认商品款和国际款
- 上传下单截图
- 创建国际批次
- 发起调价申请
- 从商品图鉴选择商品
- 管理自己的收款方式

囤货人：

- 查看分配给自己的入库和排发
- 确认入库
- 处理排发申请
- 录入国内运费
- 录入国内快递
- 确认国内运费付款
- 导出囤货工作台数据

管理员：

- 所有维护能力
- 编辑团信息
- 进入管理台
- 用户和角色管理
- 审计与异常处理
- 转单审核
- 查看审计日志

## 管理台入口

谷团详情页中应有一个管理台入口。

显示规则：

- 管理员可见
- 拼单维护人可见，但进入后只展示自己有权限的功能
- 囤货人可见时进入囤货排发工作台
- 普通成员不可见或展示为无权限

路径建议：

- `/app/groups/:groupId/admin`

说明：

- 管理台页面后续再细化。
- 第一版可以先做入口页，列出当前角色可用的管理能力。
