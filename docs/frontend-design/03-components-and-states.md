# 组件与状态规范

## 核心组件

### AppShell

用途：

- 登录后页面外壳。

包含：

- 页面标题
- 主内容区
- 底部 Tab 或 PC 侧边导航
- 当前用户上下文

### BottomTabs

用途：

- 手机端常驻底部导航。

Tab：

- 首页
- 谷团
- 我的

规则：

- 当前 Tab 高亮。
- 底部安全区留白。
- 管理台不进入底部 Tab，而是从谷团详情进入。

### GroupCard

用途：

- 谷团列表卡片。

字段：

- 团名称
- 团头像
- 成员数量
- 当前进行中的拼团数量
- 我的角色

### GroupHeaderCard

用途：

- 谷团详情顶部团信息卡片。

字段：

- 团名称
- 团头像
- 团长或维护人
- 成员数量
- 创建时间或说明
- 我的角色

操作：

- 编辑团信息
- 查看成员

### AdminEntryBar

用途：

- 谷团详情中的管理台入口。

显示规则：

- 管理员、拼单维护人、囤货人可见。
- 普通成员不可见。

文案建议：

- 标识：`工作台`
- 描述：`管理拼团、付款、转运、排发等`
- 操作：`立即进入`

### GroupBuyCard

用途：

- 拼团列表卡片。

版式：

- 上部分为主图区域，展示 `coverImageUrl` 或默认占位图。
- 下部分为信息区域，按标题、类型、状态、发起人、时间和数量信息排列。
- 手机端使用上图下信息的纵向卡片。
- PC 端可保持卡片，也可以在列表密度更高的页面中转成表格行，但字段优先级保持一致。

字段：

- 主图
- 拼团标题
- 拼团类型
- 拼团状态
- 发起人
- 截团时间
- 商品数量
- 剩余可拼数量
- 我的参与状态

操作：

- 点击进入拼团详情

### StatusTabs

用途：

- 拼团列表和拼单记录列表的状态筛选。

谷团详情筛选：

- 全部
- 进行中
- 未开始
- 已截团
- 已完成
- 已取消

拼单记录筛选：

- 全部
- 未肾
- 未补国际
- 未到货
- 可排发
- 已申请排发
- 已排发
- 异常

### SearchBar

用途：

- 列表搜索。

手机端：

- 单行搜索框
- 筛选按钮可放右侧

PC 端：

- 搜索框和筛选器横向排列

### GoodsFilterPanel

用途：

- 商品图鉴筛选详情小窗。

字段：

- 关键词
- 角色多选
- 匹配模式：包含所选角色、仅包含所选角色
- 系列或分类，后续可加

规则：

- 手机端从底部或右侧弹出，可收起。
- PC 端可以作为右侧筛选面板或浮窗。
- 筛选结果需要立即反馈数量变化或 loading 状态。

### PaymentProofDialog

用途：

- 上传付款凭证。

入口：

- 拼团详情页费用卡片
- 我的费用页费用列表

字段：

- 付款金额
- 付款时间
- 付款截图
- 备注

规则：

- 手机端使用底部弹窗。
- PC 端使用居中弹窗或右侧抽屉。
- 提交后费用状态更新为已提交。

### OrderScreenshotDialog

用途：

- 维护人上传下单截图和下单信息。

字段：

- 下单截图
- 下单人
- 下单时间
- 日本网站
- 网站订单号
- 备注
- 关联拼单商品或记录

规则：

- 日本网站和网站订单号可以为空。
- 一个拼团可以维护多个下单记录。

### SelectableRecordList

用途：

- 我的拼单、可排发商品和转单申请等需要多选的列表。

规则：

- 支持单选、多选和全选。
- 手机端选择状态要明显，操作栏可固定在底部。
- PC 端可以使用表格勾选列。

### WorkbenchModuleGrid

用途：

- 管理员工作台、囤货工作台和拼单维护人工作台的模块入口。

规则：

- 按角色展示模块。
- 模块卡片展示待处理数量、说明和进入按钮。
- 工作台页面可以提供导出按钮。

## 状态标签

### 拼团状态

- `等待开团`：中性灰
- `拼拼拼`：蓝色或绿色
- `已切`：蓝色强调
- `已截团`：灰蓝
- `已完成`：绿色
- `已取消`：灰色

### 拼单记录展示状态

- `未肾`：橙色，代表待付款
- `未转寄`：灰蓝，代表还未进入转运
- `未到货`：紫蓝或蓝灰，代表物流中
- `未补国际`：橙红，代表待补国际费
- `可排发`：绿色，代表可以操作
- `已申请排发`：蓝色
- `已排发`：绿色或青色
- `转单中`：黄色
- `已完结`：灰绿

规则：

- 状态文案必须直接使用业务词。
- 不要在 UI 上展示英文枚举。
- 如果一条记录同时满足多个不可排发原因，只展示后端返回的最高优先级状态。

## 操作按钮

主要操作：

- 手机端每个区域只突出一个主按钮。
- 列表卡片中常用操作直接展示。
- 低频操作放入更多菜单。

图标：

- 使用 `lucide-react`
- 新建使用 `Plus`
- 编辑使用 `Pencil`
- 搜索使用 `Search`
- 返回使用 `ChevronLeft`
- 进入使用 `ChevronRight`
- 上传使用 `Upload`
- 用户使用 `User`
- 首页使用 `Home`

## 表单组件

输入项：

- 文本输入
- 数字输入
- 日期时间选择
- 图片上传
- 单选或下拉选择
- 多选列表
- 多行备注

表单规则：

- 手机端表单单列。
- PC 端可以两列，但关键字段仍按业务顺序排列。
- 错误提示直接显示在字段下方。

## 空状态

常见空状态：

- 暂无拼团
- 暂无可排发商品
- 暂无付款记录
- 暂无搜索结果
- 暂无待审核转单
- 暂无异常记录
- 暂无审计日志

空状态操作：

- 有权限时显示新建按钮
- 无权限时只显示提示

## Loading 与错误

Loading：

- 页面初次加载使用骨架屏。
- 按钮提交使用按钮内 loading。

错误：

- 权限不足显示轻提示和返回入口。
- 表单错误显示字段级错误。
- 网络错误显示重试按钮。
