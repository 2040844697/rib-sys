# RibSys2 文档索引

## 当前定位

RibSys2 是一个国内拼团 goods 的全流程记录和协作系统。

它用于辅助开团、参团、下单、付款记录、国际转运、国内囤货、排发和查询。系统本身不收取费用，不代收代付，只记录线下付款和确认过程。

## 已确认业务决策

- 一个拼单可以包含多个商品
- 拼单商品在开团时以 CNY 标价，不处理汇率
- 同一成员在同一拼单内购买多个商品时，首款合并为一次“肾”
- 拼单状态第一版使用中文业务状态：等待开团、拼拼拼、已切、已截团、已完成、已取消
- 下单记录以下单截图为主要凭证，日本网站和网站订单号可为空
- 普通成员第一版可以直接在系统内选择商品参团
- 首款未确认时允许维护人先下单，但系统需要明确保留未付款状态
- 创建拼单时可以为拼主或已提前沟通成员内定库存
- 已有付款或应收后的调价需要管理审核，并通过补款或退款记录体现
- 系统支持上传付款记录和付款截图
- 国际运费通常按重量分摊
- 税费默认在本次国际运输涉及的全部订单成员中均摊，并需要支持手动编辑
- 国际费用在实际发生后再向成员生成应收
- 成员可以把多个已到国内囤货处的商品合并申请排发
- 国内运费在排发申请后单独收取，也支持到付记录
- 用户身份需要关联 QQ 号和群昵称
- 系统需要支持转单：双方先线下确认，再提交转单申请，由维护人或管理员审核通过后生效
- 第一版只有一个群组，用户与群关系可先简化；后续多群时应升级为群包含用户的成员关系
- 付款截图、收款二维码、订单截图第一版使用本地 MinIO 服务存储，业务接口仍保留 URL 抽象
- 原“打表”行为不再作为独立业务动作，归入“拼单维护人”角色
- 系统仅用于开团和记录可查，不作为收费平台

## 下一步建议

优先继续确认以下问题：

1. 费用模型是否第一版支持部分付款
2. 费用模型是否第一版支持一张付款凭证抵扣多笔费用
3. 退款是否需要像付款一样上传凭证并审核
4. 调价影响范围如何选择：仅新认领、全部未付款成员，还是全部成员
5. 已切状态下成员端只展示可拼库存，还是也展示总库存
