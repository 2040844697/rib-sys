# 角色与权限

## 设计原则

系统应服务多个视角：

- 管理员关注全局配置、权限、审计和异常处理
- 拼单维护人关注开团、下单、商品和收款记录
- 囤货人关注入库、可排发、排发申请和国内发货
- 普通成员关注自己的参团、付款、可排发商品和物流

原“打表人”不再作为独立角色出现。全流程系统中，这部分职责归入“拼单维护人”。

## 角色定义

### 管理员

管理员拥有全局管理权限。

可执行操作：

- 管理用户
- 绑定或修正 QQ 号、群昵称
- 分配角色
- 查看全部拼单、订单、费用、排发和审计记录
- 修改异常数据
- 配置群、收款渠道类型、物流公司、转运平台
- 处理争议、转单、退款、缺货等异常状态

限制：

- 管理员的关键修改必须进入审计记录

### 拼单维护人

拼单维护人负责拼单全流程中开团到国内入库前的记录维护。

可执行操作：

- 创建拼单
- 编辑自己负责的拼单
- 添加和编辑商品
- 设置商品 CNY 价格
- 为拼主或已提前沟通成员内定库存
- 上传或维护首款收款渠道
- 录入成员参团记录
- 确认首款付款
- 记录日本网站下单信息
- 更新日本订单物流状态
- 创建或维护国际转运批次
- 录入国际费用分摊
- 确认国际费用付款
- 发起调价申请
- 发起或确认转单

限制：

- 默认只能维护自己负责的拼单
- 修改已确认的付款或费用金额需要记录原因
- 已生成应收或付款后的调价需要管理审核
- 删除关键记录应使用作废，而不是物理删除

### 囤货人

囤货人负责国内到货后的入库、分货和排发。

可执行操作：

- 查看分配给自己的国际转运批次
- 确认国内入库
- 查看名下可排发商品
- 查看成员排发申请
- 拆分、核对和打包排发商品
- 录入国内运费
- 确认国内运费付款
- 录入国内快递信息
- 更新排发状态

限制：

- 默认不管理开团和日本下单信息
- 默认只能处理自己负责囤货点的排发申请

### 普通成员

普通成员使用系统查询和提交自己的信息。

可执行操作：

- 查看自己参与的拼单
- 查看自己购买的商品
- 查看自己的应收、实收和付款确认状态
- 上传付款凭证
- 查看自己可排发的商品
- 创建排发申请
- 填写或更新排发地址
- 查看国内发货物流
- 申请转单

限制：

- 只能查看自己的订单和付款记录
- 不能确认付款
- 不能直接修改已确认费用

## 用户身份

用户需要关联：

- 系统账号
- QQ 号
- 群昵称
- 所属群

QQ 号建议作为重要识别字段，但不建议作为唯一登录凭据。群昵称可能变化，应保存历史昵称或允许管理员维护别名。

## 审计记录

以下操作必须记录审计：

- 创建、修改、作废拼单
- 修改商品价格
- 创建、修改、取消内定库存
- 审核或应用调价
- 修改成员购买数量
- 确认或撤销付款
- 修改应收金额
- 调整费用分摊
- 创建或确认转单
- 确认国内入库
- 创建、修改、取消排发申请
- 录入或修改物流单号
- 管理员修改用户身份或权限

审计记录应包含：

- 操作人
- 操作时间
- 操作类型
- 业务对象类型
- 业务对象 ID
- 修改前内容
- 修改后内容
- 修改原因
